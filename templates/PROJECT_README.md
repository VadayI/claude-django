## Quick start

This project requires **WSL2 Ubuntu** (on Windows) / **Linux** / **macOS** + **Docker Desktop**. See `CLAUDE.md` → "Where this runs" for the rationale.

```bash
# 1. Fill secrets (DJANGO_SECRET_KEY, etc.)
cp .env.example .env
$EDITOR .env

# 2. Bring up Postgres + backend
docker compose up -d
docker compose ps

# 3. Apply migrations
docker compose exec backend python manage.py migrate

# 4. (Optional) Create an admin user
docker compose exec backend python manage.py createsuperuser

# 5. Open the interactive API client
# Swagger UI: http://localhost:8000/api/schema/swagger/
# Redoc:      http://localhost:8000/api/schema/redoc/
# Raw schema: docs/api/openapi.yml
```

## Process discipline

- **API-first.** Every endpoint: contract → failing test → minimal code → green → OpenAPI schema regenerated and committed in the same PR. CI drift gate enforces parity.
- **TDD.** No production code without a failing test first. Red → Green → Refactor.
- **PRs only.** `main` is protected. Branch → commits → PR → review → merge. Conventional Commits (`feat: …`, `fix: …`, `test: …`, `docs: …`, `refactor: …`, `chore: …`). Branch naming: `feat/<slug>`, `fix/<slug>`, etc.
- **Per-app README.** Every Django app under `backend/apps/<app>/` has a local `README.md` (CI gate `scripts/check_app_readmes.sh`).
- **No silent stubs.** `# STUB:` markers in production code must be logged in `docs/STUBS.md` (CI gate `scripts/check_stubs.sh`).

## Where things live

```
backend/                    # Django project (config/) + apps (apps/<domain>/)
docs/
  PROJECT.md                # project brief (fill via /synthesize-brief or by hand)
  WORKLOG.md                # what we did in each session (committed)
  STUBS.md                  # ledger of intentional # STUB: placeholders
  lessons.md                # things learned during the project
  api/
    openapi.yml             # ← source of truth for the API contract (regenerated by drf-spectacular)
    INDEX.md                # human index of endpoints / per-endpoint narrative notes
  decisions/                # ADRs — architectural decision records
  plans/                    # work plans before non-trivial features
scripts/
  detect-env.py             # writes .claude/memory/env-detect.json on SessionStart
  check_stubs.sh            # CI gate: forbids unlogged STUBs
  check_openapi_drift.sh    # CI gate: forbids code/schema drift
  check_app_readmes.sh      # CI gate: forbids apps without README
.claude/                    # agents, rules, commands, skills, memory, settings
.github/workflows/          # backend-ci.yml (ruff + gates + pytest)
docker-compose.yml          # local dev stack (postgres + backend)
```
